# Security Policy

## Reporting Security Issues

**Please do NOT report security issues through public GitHub issues.**

If you discover a security issue, please report it privately:

**Email:** security@altresear.ch

**Subject:** [8004scan Security] Brief description

### What to Include

- Description of the issue
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

### What to Expect

| Step | Timeframe |
|------|-----------|
| Acknowledgment | 24-48 hours |
| Initial Assessment | 72 hours |
| Status Update | Weekly |
| Resolution | Depends on severity |

## Scope

### In Scope

- 8004scan.io web application
- 8004scan API endpoints
- Authentication and authorization
- Data exposure issues
- Smart contract interactions

### Out of Scope

- Third-party services
- Social engineering
- Physical attacks
- Issues already reported

## Recognition

We appreciate security researchers who help keep 8004scan secure. With your permission, we will acknowledge your contribution in our security advisories.

## Safe Harbor

We will not pursue legal action against researchers who:
- Act in good faith
- Avoid privacy violations
- Do not disrupt services
- Report issues promptly
- Do not exploit issues beyond verification
