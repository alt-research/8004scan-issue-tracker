# Contributing to 8004scan

Thank you for your interest in contributing to 8004scan! This document provides guidelines for contributing to the issue tracker.

## How to Contribute

### Reporting Issues

1. **Search First**: Check if the issue already exists
2. **Use Templates**: Select the appropriate issue template
3. **Be Specific**: Provide detailed information
4. **One Issue Per Report**: Keep issues focused

### Issue Types

| Type | When to Use |
|------|-------------|
| Bug Report | Something is broken or not working |
| Feature Request | Suggest new functionality |
| Feedback | Share your experience |
| Question | Ask about usage |

### Writing Good Issues

**Do:**
- Provide clear, descriptive titles
- Include steps to reproduce (for bugs)
- Add screenshots when helpful
- Specify your environment (browser, OS, wallet)

**Don't:**
- Include sensitive data (keys, passwords)
- Submit duplicates
- Combine multiple issues in one report

## Issue Lifecycle

1. **New** - Issue submitted
2. **Investigating** - Team is reviewing
3. **In Progress** - Being worked on
4. **Resolved** - Fix deployed
5. **Closed** - Issue complete

## Community Guidelines

- Be respectful and constructive
- Follow our [Code of Conduct](CODE_OF_CONDUCT.md)
- Help others when you can
- Stay on topic

## Security Issues

Do NOT report security issues publicly. See [SECURITY.md](SECURITY.md) for instructions.
